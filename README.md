# Robotino: Robot Vacuum Cleaner Operating System

This project focuses on a modular software architecture for a modern robotic vacuum cleaner system that performs environmental sensing, navigation, mapping, and user interaction tasks.


## Project Objectives

* **Sensor Management:** Sensing and processing environmental data in real time.
* **Safe Navigation:** Implementing obstacle detection and safe movement algorithms.
* **Mapping:** Digitally mapping and recording the robot's environment.
* **User Interaction:** A user-friendly interface with an expandable submenu system.


## Technical Architecture

The software is built on the interaction of multiple classes in accordance with the principles of modularity and extensibility:

* **Modular Structure:** Each function (movement, sensor, map) is designed as an independent class.
* **UML Planning:** All classes and their interactions are planned with UML diagrams before the development phase.


## Project Evaluation

* **Achievements:** Environmental sensing, basic motion control, and mapping modules have been successfully integrated; an extensible code infrastructure has been provided. * **Areas for Improvement:** Unexpected algorithm behaviors and error handling processes encountered in specific scenarios constitute key learning points for future phases of the project.

## Proje Hedefleri

* **Sensör Yönetimi:** Çevresel verilerin algılanması ve gerçek zamanlı işlenmesi.

## Teknik Mimari

Yazılım, modülerlik ve genişletilebilirlik prensipleri doğrultusunda birden fazla sınıfın etkileşimi üzerine inşa edilmiştir:

* **Modüler Yapı:** Her fonksiyon (hareket, sensör, harita) bağımsız sınıflar olarak tasarlanmıştır.
* **UML Planlama:** Tüm sınıflar ve etkileşimleri geliştirme aşamasından önce UML diyagramları ile planlanmıştır.
